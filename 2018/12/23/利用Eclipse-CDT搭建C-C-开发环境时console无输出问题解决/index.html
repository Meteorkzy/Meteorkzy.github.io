利用Eclipse CDT搭建C/C++开发环境时console无输出问题解决

发表于 2018-12-23

这天想开始搭建一个C语言的IDE，网上看了看，看到Eclipse不错，集成度高，而且插件丰富，于
是开始着手搭建,为寒假的学习打好基础。

具体步骤：

  1. 到oracle官网下载安装jdk，然后配置环境变量。（网上教程一搜一大堆）
  2. 到Eclipse下载Eclipse CDT的package,安装。

  3. 命令行输入java -version,如果出现java版本的信息，则说明以上步骤成功。

  4. 下载C\C++的编译器MinGW，到官网上下载，我下载的是32位版本的（这也为后来我抓狂了一天埋下
    了伏笔），安装，配置环境变量。（网上教程一搜一大堆）

以上步骤做完，可以试着写几行最简单的代码测试一下了，打开Eclipse，输入以下简单的代码（代
码是《C primer plus》上的一个实例）：

完美运行，这下就开始了我漫长又痛苦的Google搜答案之旅。。。
网上方法试了个遍，都没有找到，强迫症的我，不搞清楚为什么出现这个问题绝不想善罢甘休，然后
从早上去图书馆弄到下午4点多，中午饭没吃，午觉没睡，一直在Google在折腾，期间甚至想把笔记
本电脑扔到窗外，可是，最后还是没找到方法。我放弃了，整个人晕晕沉沉地走回宿舍，吃了点东西
就上床睡到大晚上。。。

然后今天，我又开始充满精力地投入到这个问题的解决方法上了，今天思维好像比较活跃，突然就想

网上重新下载安装64位版本的，现在开始边写这篇博客边记录。

又去google了一番，没想到很快就找到答案了，居然是在Eclipse的Wiki首页找到的，，可能是昨天
太累了，看到英文文章都不想仔细看。

结果是，这好像是一个Eclipse的bug。。。（其实这个解决办法我昨天就看到了的，不过当时有点头铁，想

卒。
